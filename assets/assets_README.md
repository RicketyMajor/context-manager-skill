# Context Manager

This directory (`/context`) serves as the central brain and memory for the AI agent and the development team. It maintains a persistent state of the project's architecture, decisions, and progress to ensure seamless handoffs and continuous context.

## Structure
- `/decisions`: Contains `log.md` with a chronological record of architectural and project decisions.
- `/general`: Contains `overview.md` (project summary and rules) and `glossary.md` (key terms).
- `/handoff`: Stores session handoff documents to maintain continuity across different agent sessions.
- `/sources`: Contains `index.md`, a directory of key files, APIs, and external references.
- `/specs`: Holds individual feature specifications and requirements.
